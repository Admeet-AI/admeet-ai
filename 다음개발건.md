# 다음 개발건

## 1. 회의 종료 시 프로그레스 표시

### 현재 상태
- 종료 버튼 클릭 → `meeting:end` 전송 → 리포트 생성 API 호출 → `/report/[id]`로 이동
- 리포트 생성(`POST /api/reports/:meetingId`)이 시간이 걸릴 수 있는데, 현재 로딩 표시가 없음

### 구현 방향
- 종료 버튼 클릭 시 **풀스크린 프로그레스 오버레이** 표시
- 단계별 상태 표시:
  1. "회의를 마무리하고 있습니다..." (meeting:end 전송)
  2. "AI가 리포트를 생성하고 있습니다..." (리포트 API 호출 중)
  3. "리포트 페이지로 이동합니다" (완료 → 리다이렉트)
- 에러 발생 시 재시도 버튼 표시

### 대상 파일
| 파일 | 변경 내용 |
|------|----------|
| `apps/web/src/app/meeting/[id]/page.tsx` | `handleEndMeeting`에 상태 관리 추가 |
| `apps/web/src/components/meeting/end-meeting-overlay.tsx` | **새 파일** — 종료 프로그레스 오버레이 컴포넌트 |

## 2. AI 인사이트 패널 — 스마트 자동 스크롤

### 현재 상태
- `PersonaPanel` 내부에 `ScrollArea`로 사고/솔루션 타임라인을 표시
- AI 사고가 길어져도 자동 스크롤이 없어서 최신 내용을 보려면 수동으로 내려야 함

### 요구사항
- 새로운 사고/솔루션이 도착하면 **자동으로 하단까지 smooth 스크롤**
- **단, 유저가 직접 스크롤을 올린 상태라면 강제로 내리지 않음** (읽고 있는 내용을 방해하지 않기 위해)
- 유저가 다시 맨 아래로 스크롤하면 자동 스크롤 모드 복귀

### 구현 방향

```
isNearBottom (하단 50px 이내) → 자동 스크롤 ON
유저가 위로 스크롤 → 자동 스크롤 OFF
유저가 다시 맨 아래로 → 자동 스크롤 ON 복귀
```

- `useRef`로 스크롤 컨테이너 참조
- `onScroll` 이벤트에서 `scrollTop + clientHeight >= scrollHeight - threshold` 체크
- `isUserScrolledUp` 상태로 자동 스크롤 ON/OFF 제어
- 새 아이템 도착 시 `isUserScrolledUp`이 false일 때만 `scrollIntoView({ behavior: "smooth" })`
- 자동 스크롤 꺼진 상태에서 "↓ 새 인사이트" 플로팅 버튼 표시 (클릭하면 맨 아래로 이동 + 자동 스크롤 복귀)

### 대상 파일
| 파일 | 변경 내용 |
|------|----------|
| `apps/web/src/components/meeting/persona-panel.tsx` | ScrollArea에 스마트 스크롤 로직 추가 |
| `apps/web/src/hooks/use-smart-scroll.ts` | **새 파일** (선택) — 재사용 가능한 스마트 스크롤 훅 |

### 참고: 스마트 스크롤 훅 인터페이스 (예시)

```typescript
const { scrollRef, bottomRef, isAtBottom } = useSmartScroll({
  deps: [timeline.length], // 이 값이 변하면 스크롤 시도
  threshold: 50,           // 하단 판정 여유값 (px)
});
```

## 3. 회의록 `/report` 페이지 — 마크다운 렌더링 리디자인

### 현재 상태
- `<pre className="whitespace-pre-wrap font-mono">`로 마크다운 원문을 그대로 출력
- 제목, 리스트, 볼드, 코드블록 등 마크다운 문법이 렌더링되지 않음
- 카드 하나에 전체 텍스트가 들어가 있어 시각적 구분이 없음

### 요구사항
- 마크다운을 **HTML로 파싱**하여 예쁘게 렌더링
- 랜딩페이지 디자인 톤과 일관된 타이포그래피
- 섹션별 시각적 구분 (배경, 보더, 아이콘 등)
- 인쇄/PDF 내보내기 시에도 깔끔하게 보일 것

### 구현 방향

#### 라이브러리 선택
```bash
npm -w apps/web install react-markdown remark-gfm
```
- `react-markdown` — 마크다운 → React 컴포넌트 변환
- `remark-gfm` — 테이블, 취소선, 체크리스트 등 GitHub Flavored Markdown 지원

#### 커스텀 컴포넌트 매핑
```tsx
<ReactMarkdown
  remarkPlugins={[remarkGfm]}
  components={{
    h1: ({ children }) => <h1 className="text-2xl font-bold ...">{children}</h1>,
    h2: ({ children }) => <h2 className="text-xl font-semibold border-b ...">{children}</h2>,
    h3: ({ children }) => <h3 className="text-lg font-semibold ...">{children}</h3>,
    ul: ({ children }) => <ul className="list-disc pl-6 space-y-1 ...">{children}</ul>,
    ol: ({ children }) => <ol className="list-decimal pl-6 space-y-1 ...">{children}</ol>,
    blockquote: ({ children }) => <blockquote className="border-l-4 border-[#00d4ff] ...">{children}</blockquote>,
    table: ({ children }) => <div className="overflow-x-auto"><table className="w-full border-collapse ...">{children}</table></div>,
    code: // 인라인 코드 vs 코드블록 분기
  }}
/>
```

#### 페이지 레이아웃 개선
- 상단: 회의 제목 + 날짜 + 참여자 정보 헤더
- 본문: prose 스타일 마크다운 렌더링 영역
- 하단: 복사/다운로드/새 회의 버튼
- 사이드: TOC(목차) 네비게이션 (선택)

### 디자인 참고 (랜딩페이지 톤)
- 배경: `bg-[#f8f9fb] dark:bg-[#0a0a0f]`
- 카드: `border-slate-200/60 dark:border-white/[0.06] bg-white/60 dark:bg-white/[0.02] backdrop-blur-sm`
- 제목 강조: `bg-gradient-to-r from-[#0066ff] via-[#00d4ff] to-[#7c3aed] bg-clip-text text-transparent`
- 인용문 보더: `border-[#00d4ff]`

### 대상 파일
| 파일 | 변경 내용 |
|------|----------|
| `apps/web/src/app/report/[id]/page.tsx` | 마크다운 렌더링으로 전면 리디자인 |
| `apps/web/src/components/report/markdown-renderer.tsx` | **새 파일** — 커스텀 마크다운 컴포넌트 매핑 |
| `package.json` | `react-markdown`, `remark-gfm` 의존성 추가 |
